Каждый член команды удовлетворяет свои потребности во время путешествия. У каждого свои потребности, иногда они совпадают с потребностями других членов команды.

Каждый член команды может тянуть проект в каком-то направлении. Если вектора направления большинства членов команды совпадают, значит команда слаженная. (Лебедь, рак и щука).

Даже имея разные потребности, членам команды может быть по пути. Лучше собирать команду из людей, которым по пути.

Команда - это партнёры, организация это партнёры. Все, что люди делают - добровольно и по желанию. Никто никому не имеет право приказывать и ожидать безусловной исполнительности. Иерархия может быть только ролевая, а не личностная. Менеджеры занимаются организацией процесса а не принуждением.

Если ключевые задачи распределять на менеджера, он будет уставать, плохо менеджерить и плохо ключевые задачи делать или долго, что блочит остальных. У нас это и происходило. Я много на себя взял и пацыки прогеры долго ниче не делали.

Ещё важным условием движения проекта является то, что у него должен быть ненулевой вектор скорости. Для этого нужно, чтоб у членов команды реализаторов был свой вектор скорости, и очень хорошо, когда он у каждого не нулевой и направлен по намеченному направлению.
Каждому участнику проекта для понимания того, правильно ли направлен его вектор движения, нужно координироваться с командой: с менеджером и с другими членами команды. Роль менеджера в том, чтоб, создавать и реализовывать план, намечать общее направление, координировать команду и направлять каждого члена команды в направлении соответствующем его роли. Важно чтоб члены команды вводили менеджера в курс дела и убеждались в том, что они двигаются туда, куда надо.
